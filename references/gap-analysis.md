# 甲方需求差距分析工作流

当用户提供甲方/客户的功能需求清单，要求与现有系统对比找出差距时，使用本工作流。

## 触发

- "对标甲方要求交付功能清单，看看少哪些"
- "对比一下XX需求和现有系统"
- "甲方给了这个清单，看哪些有了哪些没有"
- "这版需求跟现在系统差多少"

## 三步法

### Step 0: 解析甲方交付清单（新增）

甲方往往以 **Excel (.xlsx)** 形式提供需求清单。使用 openpyxl 提取：

```python
import openpyxl
wb = openpyxl.load_workbook('甲方需求清单.xlsx')
ws = wb.active
for row in ws.iter_rows(values_only=True):
    # 每行: [功能分类, 功能描述, ..., 开发成本(人天)]
```

**典型Excel结构：**
- Sheet1: 模块1（小程序）功能清单 + 模块2（服务平台）功能清单 + 报价行
- 每行包含：功能分类/功能描述/人天数
- 最后有合计行和报价说明行

**坑：** Excel中功能描述可能被单元格截断（只能看到前120字符）。用 `str(v)[:200]` 而非默认的 `[:60]` 确保完整提取。

### Step 1: 逐项标注状态

将甲方需求清单的每一项，标注三种状态：

| 标记 | 含义 | 判断标准 |
|:----|:-----|:--------|
| 🟢 已覆盖 | 现有系统完全支持 | 字段/功能在现有系统中对应明确 |
| 🟡 部分覆盖 | 有基础但不够完整 | 有对应模块/字段但缺少关键能力 |
| 🔴 完全缺失 | 现有系统没有 | 需要从零新建 |

**关键原则：🟡项要标注"现有系统对应什么"和"还缺什么"两个维度。**

### Step 2: 分类汇总

按模块分组统计，产出总览表格：

```
| 模块 | 总数 | 🟢 | 🟡 | 🔴 |
```

### Step 3: 输出HTML报告

报告结构：

1. **顶部渐变Header** — 靛紫渐变 + 产品名称 + 日期
2. **毛玻璃统计卡片** — 四列：总需求数 / 已覆盖 / 部分覆盖 / 完全缺失
3. **逐模块可折叠卡片** — 每个子模块一个折叠卡片，内部是完整表格
4. **三期交付建议** — 三列卡片：一期补齐 / 二期新建 / 三期智能化

## HTML报告设计规范

使用靛紫渐变毛玻璃风格（与原型设计系统一致）：

```css
--primary: #6366F1; --primary-dark: #4F46E5;
--bg: linear-gradient(135deg, #eef2ff, #e0e7ff, #f0f2ff);
--card-bg: rgba(255,255,255,0.85);
backdrop-filter: blur(16px);
```

**卡片折叠策略：** 模块卡片默认折叠（`collapsed` class），让报告首屏展示全局概览而非淹没在细节中。用户可逐模块展开查看。

**状态标签配色：**
- `tag-red`: `#FEF2F2` 背景 + `#EF4444` 文字 + `#FECACA` 边框
- `tag-yellow`: `#FFFBEB` 背景 + `#92400E` 文字 + `#FDE68A` 边框
- `tag-green`: `#ECFDF5` 背景 + `#065F46` 文字 + `#A7F3D0` 边框

**表格规范：**
- 四列：序号 / 功能点 / 状态(标签) / 现有系统对应
- 表头 `rgba(99,102,241,0.05)` 背景 + 下划线
- 行hover `rgba(99,102,241,0.02)`
- 斑马纹不使用（已覆盖/缺失行颜色本身就是视觉区分）

## 示例输出

参考本次会话产出的 `/home/kk/汽拆系统PRD/甲方需求差距分析报告.html`：
- 89项总需求，28已覆盖，14部分覆盖，47完全缺失
- 模块1（小程序）35项全部缺失
- 模块2（服务平台）54项混合状态
- 三期计划：一期补齐→二期小程序→三期智能化
